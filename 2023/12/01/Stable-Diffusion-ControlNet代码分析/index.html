Overview

ControlNet 是基于 Stable Diffusion 的代码库开发的，以下为整个仓库的文件组织结构（排除了非代码文件，如文档、字体、测试图片等）：

Details

annotator

Annotator 指各种条件提取器，如 canny 边缘检测器、midas 深度和法线估计模型、openpose 人体姿态识别器等，存放在 annotator 下。这些 annotator 既有传统方法，也有基于深度学习的方法，因此有些 annotator 需要模型权重文件，权重文件应放置在 annotator/ckpts 下。

作者将每一种 annotator 分别放在对应子目录下，并且在各自的 __init__.py 中封装了 xxxDetector 类，例如 CannyDetector、MidasDetector、OpenposeDetector 等。这些 xxxDetector 有 __call__() 方法，接受输入图像（numpy uint8 HWC），返回检测图像（numpy uint8 HWC），因此调用起来非常的方便。

作者还在 annotator/util.py 中提供了两个工具方法：resize_image() 将图像按小边成比例缩放到指定大小附近的 64 倍率处，HWC3() 将不同通道数的图像统一为 3 通道。

ldm/modules

ldm/modules 下是各种网络模块的定义（即继承 torch.nn.Module 的类）。

ldm/modules/ema.py

该文件定义了 LitEMA 类，方便在训练过程中维护 EMA 权重。

  • 初始化时传入 model，则 LitEMA 会将其参数存入自己的 buffer；
  • 每次调用 LitEMA 时传入新的 model，则 LitEMA 更新自己的 buffer；
  • copy_to() 函数把自己的 buffer 复制给传入的 model；
  • store() 函数将传入的参数暂存起来；
  • restore() 函数将暂存的参数赋值给传入的参数。

ldm/modules/attention.py

该文件定义了一些注意力模块，包括：

  • CrossAttention：交叉注意力模块。输入特征 x、条件 context 和掩码 mask，输出与 x 维度相同。如果 context=None，则等价于自注意力。
  • MemoryEfficientCrossAttention：利用 xformers 包做交叉注意力，功能与 CrossAttention 相同，但更省内存。
  • BasicTransformerBlock：SelfAttention + CrossAttention + FF 串起来。输入特征 x 和条件 context，输出与 x 维度相同。
  • SpatialTransformer：支持图片形式输入输出的 BasicTransformerBlock 列表。输入特征图 x 和条件 context，输出与 x 维度相同。

ldm/modules/encoders

这里的 encoders 指各种文本编码器。ldm/modules/encoders/modules.py 在 transformer 包和 openclip 包的基础上进行了进一步封装，包括：

  • FrozenT5Embedder
  • FrozenCLIPEmbedder
  • FrozenOpenCLIPEmbedder
  • FrozenCLIPT5Encoder

它们都有 encode() 方法，调用即可获取输入文本的 embeddings.

Stable Diffusion v1.5 使用的是 FrozenCLIPEmbedder，加载的权重是 openai/clip-vit-large-patch14.

ldm/modules/distributions

ldm/modules/distributions/distributions.py 定义了各种概率分布，包括：

  • DiracDistribution：狄拉克分布
  • DiagonalGaussianDistribution：各分量独立高斯分布

它们都有方法 sample() 和 mode() 用于采样以及返回均值。高斯分布还有 kl() 和 nll() 计算 KL 散度和负对数似然。

ldm/modules/diffusionmodules

ldm/modules/diffusionmodules 下是搭建 VAE 和 UNet 的各种模块。

  • model.py：仿照 UNet 设计的各种 Encoder 和 Decoder；
  • openaimodel.py：参考自 openai 代码的 UNetModel. 其 forward() 接受噪声图 x、时间步 timestep、以交叉注意力融入的条件 context 和类别标签 y（即以 adm 的方式融入模型的条件，不一定真的表示类别）。

Stable Diffusion v1.5 使用的 UNet 就是 openaimodel.py 中定义的 UNetModel；使用的 VAE 就是 model.py 中定义的 Encoder 和 Decoder.

ldm/models

上文中 ldm/modules 目录下定义了所有 SD 需要用到的网络模块，但由于 Stable Diffusion 项目（包括 ControlNet 项目）采用的是 PyTorch Lightning 训练，因此还需要对接 PL，这就是 ldm/models 的作用。

PL 要求定义一个继承 pl.LightningModule 的类，其中不仅要在 __init__() 中实例化各个 nn.Module 模块组件，还要重写一些方法。在 Stable Diffusion 项目中，重写的方法包括：

  • configure_optimizers(self)：返回一个元组，第一个元素为 optimizers 列表，第二个元素为 schedulers 列表。
  • training_step(self, batch, batch_idx)：参数为一个 batch 的数据和当前 batch 的编号，返回一个 loss 张量或一个含有 loss 字段的字典。
  • validation_step(self, batch, batch_idx)：参数为一个 batch 的数据和当前 batch 的编号，返回一个 loss 张量或一个含有 loss 字段的字典。
  • on_train_batch_end(self, outputs, batch, batch_idx)：这是一个 hook，用于在一个 batch 训练结束后执行里面的内容。在 Stable Diffusion 项目中用于更新 EMA 权重。
  • log_images()???

ldm/models/autoencoders.py

该文件定义了 SD 中的 VAE 模块，主要实现了 AutoencoderKL 类（继承 pl.LightningModule）。

该类包含四个 nn.Module 组件：

  • encoder：Encoder 类的实例（上文提及），即编码器；
  • decoder：Decoder 类的实例（上文提及），即解码器；
  • quant_conv：一层 1x1 卷积 nn.Conv2d，将编码器隐空间维度 (=512) 映射到 embedding 维度 (=4)；
  • post_quant_conv：一层 1x1 卷积 nn.Conv2d，将 embedding 维度 (=4) 映射回解码器隐空间维度 (=512)。

forward() 流程为：encoder 编码均值方差 → quant_conv 降维 → 从服从该均值方差的高斯分布中采样 → post_quant_conv 升维 → decoder 解码输出图像。

ldm/models/diffusion

该目录下定义了扩散模型的 PL 接口以及各种采样器。

首先来看采样器。项目实现的采样器包括：

  • DDIMSampler：在 ldm/models/diffusion/ddim.py 中；
  • PLMSSampler：在 ldm/models/diffusion/plms.py 中；
  • DPMSolverSampler：在 ldm/models/diffusion/dpm_solver 目录下。

这些采样器都是普通的类，有 sample() 方法，给定采样步数、batch size、图像大小以及其他参数，就能迭代地从高斯噪声生成最终的图像。


还剩下一个 ddpm.py 是整个仓库里最长的文件，有 1800 行，其中定义了 DDPM 类、LatentDiffusion 类、DiffusionWrapper 类以及一系列继承自 LatentDiffusion 来做微调的类。我们着重看前三个。

DiffusionWrapper 类继承自 pl.LightningModule，其主要作用是将不同种类的条件以不同的方式融入扩散模型，包含一个组件 diffusion_model，是上文提到的 UNetModel 的实例。

其 forward() 函数接受带噪图像 x、时间步 t、条件 c_concat、条件 c_crossattn和条件 c_adm. 顾名思义，后面三个条件的区别在于融入模型的方式不同，融入哪些条件由参数 conditioning_key 决定。回忆上文提到 UNetModel 的 forward() 本身支持两种条件——context 表示交叉注意力、y 表示 adm 方式，因此 DiffusionWrapper 的 forward() 就是基于 UNetModel 的 forward() 上的进一步封装。这个类应该是为了适应更多种类的条件而后来添加的，否则直接在 UNet 的 forward() 函数中处理更好。

DDPM 类也继承自 pl.LightningModule，包含一个组件 model，是 DiffusionWrapper 的实例。

LatentDiffusion 类是 SD 最主要的类。它继承自 DDPM 类，在其基础上增加了两个组件：

  • cond_stage_model：FrozenCLIPEmbedder 类（见下文）的实例，即文本编码器；
  • first_stage_model：AutoencoderKL 类的实例，即 VAE.

cldm

上面 ldm 里的一大堆其实都是 Stable Diffusion 的工作，而 cldm 里才是 ControlNet 相关的代码。让我们看看作者是怎么把 ControlNet hack 进 Stable Diffusion 中的。

cldm/cldm.py

这里是 ControlNet 的核心，定义了三个类：ControlledUNetModel、ControlNet 和 ControlLDM.

ControlledUNetModel 继承自 UNetModel（即 SD 的去噪 UNet），重写了 forward() 方法，从而把 ControlNet 的输出接入 SD 之中。实现上也没有什么 tricky 的地方，就是老老实实抄过来改写。

ControlNet 就是 ControlNet 本体了，是一个 nn.Module 模块。其大部分内容就是把 UNetModel 的 Encoder 部分抄过来，但是需要新添加一个将条件输入压缩到隐空间维度的 8 层卷积，称作 input_hint_block. 另外还加了一些 zero_convs.

ControlLDM 则继承自 LatentDiffusion，提供 PyTorch Lightning 的接口。其新增了组件 control_model，即 ControlNet 的实例化。

cldm/ddim_hacked.py

该文件主要修复了 ldm/models/diffusion/ddim.py 在采样时的一个 bug，可以直接平替原文件使用。

gradio_xxx.py

基于 gradio 包制作 webui，每种条件一个 webui. 下面只以 gradio_canny2image.py 为例进行分析。

处理函数接收以下输入：

  • input_image：numpy uint8 HWC 格式
  • prompt：文本
  • a_prompt：额外文本
  • n_prompt：负向文本
  • num_samples：采样数量
  • image_resolution：图像分辨率
  • ddim_steps：DDIM 采样步数
  • guess_mode：是否使用 guess mode
  • strength：ControlNet 加入的强度
  • scale：CFG scale
  • seed：随机种子
  • eta：DDIM eta
  • low_threshold：canny low threshold
  • high_threshold：canny high threshold

处理函数的处理流程在代码里清晰明了，这里就不多赘述了。值得一提的是，作者没有给出非 webui 的推理脚本，而我在使用的时候发现 gradio 的版本似乎存在一些问题。不过根据 gradio 代码，稍微改改写一个非 webui 的推理脚本也并不难。
